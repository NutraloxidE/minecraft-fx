# 【完全版】Admin から実行停止できる裁定取引のサーバー管理スクリプト

## 目的

* 複数のアイテムペアを対象に、価格差を機械的に検出して裁定取引（アービトラージ）を行う
* 管理者が任意のタイミングで実行停止できるようにする

## アカウント選択

* 管理画面で、裁定取引に使用するアカウント（プレイヤーまたはシステムUUID）を選択できる

## 実行条件

* アイテムペアが2つ以上存在する場合のみ実行する
* 全ペアを対象に監視し、条件を満たしたペアから順に裁定取引を行う
* 基本の検査間隔は15秒とする

## 🔄 サーバーサイドの並行処理仕様（Paper MC API 最適化）

* Java標準の `ScheduledExecutorService` ではなく、Paperのライフサイクルに完全に同期した非同期タスク（`Bukkit.getScheduler().runTaskTimerAsynchronously` または Paperの `ServerScheduler`）を使用して15秒のループを実行すること
* プラグインの無効化（`onDisable`）時に、このタスクが安全にキャンセル（`cancel()`）され、ゾンビスレッドやデータの不正書き込みが残らない堅牢な設計にすること

## 発注条件

* ネットワーク通信（HTTP Fetch）は一切経由せず、プラグイン内部のサービス層（`OrderService` や `MatchingEngine` など）の注文執行関数を直接コールする（インプロセス直接実行）
* 機械的な計算に基づき、価格差が十分に開いた場合のみ発注する
* 最後に設定したリミット価格が、直近の最終約定価格から離れすぎている場合はキャンセルする
* 注文時は、選択されたアカウントのUUIDとして内部処理を通すことで、一般プレイヤーと全く同じバリデーション、残高チェック、Taker手数料（0.1%〜0.16%）の徴収ロジックを例外なく通過させること

## スリップ判定

* 監視中に、機械的な範囲で取引操作の兆候（急激な板の減少や価格変動）が検知された場合は、その回の取引を行わない
* この場合はスリップ扱いとし、発注を見送る

## 停止とステート管理

* Admin画面から実行中の裁定取引を停止できる
* Admin API側（`/api/admin/arbitrage/toggle`等）は、このスクリプトの実行フラグ（`boolean enabled`）や選択されたアカウント情報をメモリ上のシングルトンまたは設定（yml/DB）に書き換えるだけの最小限の実装とし、HTTPクライアントなどをプラグイン内部に新設しないこと
